### 2.2 前端要求
- **必须直接使用该 HTML 文件作为用户端唯一入口**，不得重新设计布局。  
- 所有动态数据（奖池、倒计时、余额、购买记录、赢家墙等）通过 JavaScript 调用后端 API 填充。  
- 多语言切换功能：HTML 中已预留 `data-i18n` 属性，需对接后端语言包或前端 JSON。  
- WebSocket 或轮询更新倒计时、奖池。  
- 该 HTML 已包含所有必要的 CSS 类名、ID，后端 API 返回的数据结构必须与之兼容（见第6节）。  

### 2.3 改动限制
- 允许在 HTML 头部添加 `<script>` 用于初始化 API 调用，但不得修改 DOM 结构。  
- 所有交互（购买、提现、切换语言、复制链接等）必须保留原有设计。

## 5. 可视化后台管理面板（Admin）详细设计

后台独立于用户端，使用不同的路由（如 `/admin`），需登录认证（超级管理员 + 操作员角色）。

### 5.1 仪表盘（Dashboard）
- 实时数据卡片：  
  - 当前奖池金额、倒计时剩余秒数  
  - 今日充值总额（USDT）、今日提现总额、今日净流入  
  - 当前在线用户数（WebSocket 连接数）  
  - 总注册用户数、首充用户数  
- 折线图：近7天充值/提现趋势  
- 轮次列表：最近10轮（ID、赢家、奖池金额、结算时间）  

### 5.2 游戏参数配置（傻瓜式开关）
- **基础参数**（滑块 + 数字输入框）：  
  - 初始奖池（BF）  
  - 倒计时秒数  
  - 最低购买金额  
  - 赢家百分比（默认70）  
  - 项目方抽成百分比（自动计算补全至100）  
- **高级参数**（可折叠）：  
  - 是否允许机器人购买（开关）  
  - 机器人购买间隔（秒）  
  - 机器人金额范围（最小/最大）  
  - 是否强制冷启动模式（开关）  
- **一键应用**按钮：修改后立即生效，无需重启服务。

### 5.3 用户管理
- 用户列表：展示用户ID、昵称、钱包地址、余额、注册时间、邀请人、状态（正常/冻结）。  
- 搜索：按 ID/昵称/地址。  
- 操作：冻结/解冻、手动增减余额（需备注原因）、重置密码（连接钱包模式不需要）。  
- 查看用户详情：购买历史、提现记录、邀请树。

### 5.4 订单与支付管理
- 充值订单列表：订单号、用户、金额、链/法币、状态、创建时间。  
- 手动标记支付成功（应急用，有权限控制）。  
- 支付网关配置：填入 API Key、回调地址等（加密存储）。

### 5.5 提现审核
- 提现申请列表：待审核、已通过、已拒绝。  
- 审核操作：通过（自动打款或标记手动打款）、拒绝（返还余额）。  
- 批量通过（仅限小额）。  
- 查看提现地址，支持一键复制。

### 5.6 机器人/假数据配置
- **机器人购买**：开关、频率、金额范围、是否允许成为赢家（默认不允许）。  
- **前端假数据**：开关“伪造今日活跃球迷”、“伪造已提现总额”、“伪造赢家墙”（这些仅前端展示，不进入后端逻辑）。  
- **手动触发**：立即执行一次机器人购买。

### 5.7 安全与风控设置
- 全局 API 限流：每 IP 每秒请求数、每用户每秒购买次数。  
- 验证码开关（reCAPTCHA）。  
- 邀请防刷：同一 IP 每日最多邀请数、设备指纹检测开关。  
- 提现风控：最低购买次数要求、购买后冷却时间（小时）。  

### 5.8 多语言管理
- 查看/编辑所有语言包（JSON 编辑器）。  
- 新增语言。  
- 默认语言设置。

### 5.9 日志与监控
- 操作日志：管理员所有操作记录（修改参数、审核提现、冻结用户）。  
- 系统日志：错误日志、异常结算记录。  
- 导出日志（CSV）。

## 6. 后端 API 设计

与 Stitch 前端 HTML 完全匹配。需实现以下核心接口（示例）。

### 6.1 游戏状态
GET /api/game/state
Response:
{
"prizePool": 125430,
"countdown": 58,
"roundId": "uuid",
"lastPurchase": { "nickname": "Messi", "amount": 10, "timeAgo": "5s" }
}


### 6.2 购买
POST /api/game/purchase
Request: { "amount": 10 }
Response: { "success": true, "newBalance": 240, "newPrizePool": 125440 }
Error: { "error": "INSUFFICIENT_BALANCE" }


### 6.3 用户信息
GET /api/user/profile
Response: { "balance": 250, "inviteCode": "abc", "invitedCount": 3, "commission": 0 }


### 6.4 充值创建
POST /api/payment/create
Request: { "amount_usdt": 10, "chain": "TRC20" }
Response: { "paymentId": "px_123", "qrCodeUrl": "...", "expiresAt": "..." }


### 6.5 提现申请
POST /api/withdraw
Request: { "amount_usdt": 50, "address": "..." }
Response: { "withdrawId": "...", "fee": 2.5, "net": 47.5, "status": "pending" }


### 6.6 最近购买记录（滚动列表）
GET /api/game/recent-purchases?limit=20
Response: [ { "nickname": "...", "amount": 5, "timeAgo": "10s" } ]


### 6.7 赢家墙
GET /api/game/winner-wall?limit=10
Response: [ { "nickname": "...", "amount": 12500, "timeAgo": "2m" } ]


### 6.8 多语言包
GET /api/locales/{lang}.json
Response: { "app_title": "闪电终局", ... }
